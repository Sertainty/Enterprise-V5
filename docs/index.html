Data that protects itself

Sertainty Enterprise is the Server Framework — everything in the Core Framework, plus an extensible web-service layer, a services database, automated workflows, and multi-user management. Active intelligence protects the data; the server centralizes how that protection is run, audited, and delegated across the organization.

Server management · five managed services

Core Services

Shared metadata, user management, and journaling across the deployment.

Data Service

Server-side UXP Object & Identity creation, plus an event-tracking registry.

Delegate Service

Remote user-access management — authorize & revoke users for UXP Objects.

Utility Service

General-purpose helpers for server-side scripting (file, string, XML).

Workflow Manager

Automated file-watching task pipelines for hands-off protection.

Key capabilities

Selective & partial access

  • Who can read what — selective access governs page-level reach.
  • How much is unprotected — only the portion in use, at runtime.
  • How long plaintext exists — decrypted just in time, released after use.

Active Intelligence, Active Authorization

Every open triggers a fresh evaluation against the embedded rules: identity, time, location, device, and prior failures.

Access is never unconditional.

Centralized auditing & multi-user management

The services database (SQLite or MySQL) journals access events and manages users centrally across the deployment.

No-code utility applications

Enterprise ships utility apps — Tech Hub (with server management) and more — that create Identities, protect files, and run operations without writing code.

Offline by default. Airgap ready.

  • No server, key manager, or network is needed for access decisions.
  • UXP runs identically in connected and airgapped environments.
  • Network features are opt-in.

Event log & SIEM

Logging is a preference set by the data owner. When the UXP is online, access events can be sent to a server through several outbound methods, ready for ingest by a SIEM or other monitoring tool.

Web-service API

An extensible HTTP layer (SertaintyHttpServer) exposes UXP operations; each message is sealed with AES-256 and a fresh 12-byte random IV per request.

Self-destruct & Alternate Reality

Configurable defensive responses when access conditions fail: shred the contents, expose decoy data, or revoke the file silently.
